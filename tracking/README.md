## Promotion rule

A package moves to **Launch Ready** only when:

1. It completes inside the target production week.
2. Actual labor remains inside the package economics.
3. No critical QA defect remains open.
4. Source, runtime, preview, manifest, QA, and handoff folders are complete.
5. A reviewer can open and use the package without a meeting.
6. Actual hours, defects, revisions, and packaging time are recorded here.
