The source project is a fan-made browser simulator. Its reusable value is not the copyrighted setting or sigil vocabulary. Its value is the production architecture:

```text
freehand input
→ deterministic cleanup
→ spatial classification
→ dictionary recognition
→ parser AST
→ behavior IR
→ diagnostics
→ live rendering
```

## What the studio absorbed

### 1. Creative actions become compilable source

The drawing is not treated as a flat image. It becomes structured source code. The studio evolves this into a generic visual production circuit whose symbols compile to asset operations.

### 2. Parser truth stays separate from executable behavior

`StudioGlyphAST` owns observed geometry and recognition. `StudioIR` owns ordered production behavior. The surgical builder owns mutation and evidence. Keeping those contracts separate prevents UI guesses from leaking into production execution.

### 3. Dictionaries combine recognition and semantics

A shape dictionary describes both what a mark looks like and what it means. The first studio dictionary includes input, normalization, palette enforcement, contact sheets, Godot compilation, packaging, strictness, evidence, and parallel intent.

### 4. Diagnostics are a first-class product surface

Unknown marks, open scope boundaries, missing inputs, duplicate operators, missing execution bindings, and stale builder state remain visible. The system does not silently substitute behavior.

### 5. Prepared and active states matter

An open scope ring is a prepared composition. Closing the boundary attempts compilation. This creates a useful review gesture: design freely while open, then close to request executable validation.

### 6. Labs become capability factories

The source project includes template, detector, and effect labs. The studio counterpart should grow dedicated labs for operator authoring, palette tuning, temporal alignment, Godot resources, and package inspection.

## What the studio deliberately did not absorb

- Witch Hat Atelier names, canonical spell rules, glyphs, artwork, or visual effects
- fan-project identity or presentation
- assumptions that a single ring or one primary symbol is sufficient for all production graphs
- renderer behavior that cannot be validated against studio delivery contracts

## Evolution path

1. **Visual compiler foundation — implemented**
   - freehand capture
   - scope detection
   - primitive operator recognition
   - `StudioGlyphAST`
   - `StudioIR`
   - diagnostics and live flow preview
   - surgical-builder plan export

2. **Production operators — implemented**
   - image normalization
   - palette enforcement
   - contact sheets
   - Godot `SpriteFrames`
   - deterministic ZIP packages

3. **Next evolution**
   - dictionary-authored templates rather than fixed primitive rules
   - nested subcircuits and reusable production macros
   - graph scheduling and content-addressed incremental builds
   - browser-to-local builder bridge with preview/apply approval
   - temporal frame alignment and stable-region constraints
   - model suggestions represented as reviewable candidate circuits, never direct writes
   - replayable sessions and comparison of alternative compositions
