# Change Control

## A change request is any request that alters

- Asset quantity
- Character or scene count
- Animation duration or frame count
- Approved style
- Target platform or engine version
- Number of formats or aspect ratios
- Delivery date
- Revision count
- Rights, security, or confidentiality requirements

## Change handling

1. Record the request.
2. Classify it as defect, clarification, or scope expansion.
3. Correct defects within the original package.
4. Handle clarifications only when they do not increase production effort materially.
5. Price scope expansion as an add-on or follow-on sprint.
6. Update schedule and acceptance criteria before beginning expanded work.

## Default add-on categories

- Additional primary asset
- Additional animation loop
- Additional VFX treatment
- Additional platform format
- Second revision round
- Rush turnaround
- New style direction
- Additional model or adapter evaluation
- Integration into a second engine or product
